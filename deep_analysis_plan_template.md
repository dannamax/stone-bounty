# Deep Analysis Plan Template

## Phase 1: Issue Requirements Deep Dive
- [ ] Read the issue description multiple times
- [ ] Identify explicit requirements vs implicit expectations  
- [ ] Analyze issue context and conversation history
- [ ] Determine success criteria and acceptance conditions
- [ ] Identify potential edge cases and boundary conditions

## Phase 2: Codebase Archaeology
- [ ] Explore existing codebase structure and patterns
- [ ] Identify relevant files and modules that need modification
- [ ] Analyze existing test patterns and coding conventions
- [ ] Understand data flow and architecture constraints
- [ ] Document current implementation limitations

## Phase 3: Solution Design & Validation
- [ ] Propose multiple solution approaches with trade-offs
- [ ] Validate solution against existing codebase patterns
- [ ] Ensure compatibility with existing functionality
- [ ] Plan comprehensive test coverage strategy
- [ ] Identify potential failure points and mitigation strategies

## Phase 4: Implementation with Quality Assurance
- [ ] Implement solution following TDD principles
- [ ] Write comprehensive unit and integration tests
- [ ] Perform static analysis and code quality checks
- [ ] Validate against original issue requirements
- [ ] Document implementation decisions and rationale

## Phase 5: Pre-Submission Review
- [ ] Self-review implementation against issue requirements
- [ ] Verify all tests pass and cover edge cases
- [ ] Check for code style and convention compliance
- [ ] Ensure documentation is complete and accurate
- [ ] Prepare comprehensive PR description with validation evidence
