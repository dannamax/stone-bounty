# Deep Analysis Plan Template

STATUS: DRAFT

## 目标 / DoD
- 目标：深入理解并高质量完成 GitHub issue
- 完成定义（DoD）：PR 被维护者接受，无驳回

## 范围
- 做：
  - 完整的需求分析和理解
  - 深入的代码库分析
  - 全面的技术方案设计
  - 充分的测试和验证
  - 高质量的 PR 提交
- 不做：
  - 表面化的实现
  - 缺少测试的代码
  - 不符合项目规范的提交

## 深度分析里程碑

### M1: 需求理解分析
- 输入：GitHub issue 详情、评论、标签
- 输出：需求分析报告、验收标准清单
- 验收标准：完全理解 issue 的真实需求和业务目标

### M2: 代码库深度分析  
- 输入：相关代码库、现有实现、编码规范
- 输出：代码库分析报告、技术约束清单
- 验收标准：深入了解项目结构和实现模式

### M3: 技术方案设计
- 输入：需求分析 + 代码库分析结果
- 输出：详细技术方案、风险评估、测试策略
- 验收标准：方案全面考虑各种场景和边界情况

### M4: 实现和测试
- 输入：技术方案、测试策略
- 输出：完整实现代码、全面测试用例
- 验收标准：代码通过所有测试，符合项目规范

### M5: 提交前审查
- 输入：实现代码、测试结果
- 输出：PR 内容、审查报告
- 验收标准：PR 准备就绪，预期会被接受

## 风险 & 回滚
- 风险：
  - 需求理解偏差
  - 技术方案不完整
  - 测试覆盖不足
  - 维护者期望不符
- 回滚策略：
  - 保留每个里程碑的中间产物
  - 记录所有决策和假设
  - 准备快速迭代和修改
